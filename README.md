# 三国英杰传 · 网页版

单机战棋试玩、本地/云端存档与账号系统（注册登录 + JWT）。

## 已实现玩法

### 关卡与单位

- **刘备线战役**（演义向）：共 **9 关**，按顺序推进——序章黄巾 → 洛阳/西凉 → 徐州 → 小沛 → 新野 → 长坂 → 赤壁前哨 → 剑阁 → 汉中；配置集中在 `client/src/game/scenarios.ts`。地图随关变大（约 **16×10** 至 **24×14**），地形与敌军规模、等级随剧情抬升；场上可出现 **演义名将**（数据来自 `client/src/game/generals.ts` 图鉴）。
- **我军**：刘备、关羽、张飞（各有 HP、武力/智力/防御、移动力、计策等）；胜利进入下一关时会 **继承** 存活武将的等级与属性并重置兵力/计策。
- **胜负流转**：多数关卡需 **全歼敌军**；部分关卡为 **击破主将**（如张角、吕布、刘璋、夏侯惇本队）即可胜利，战报与侧栏会写明。最后一关胜利后从 **序章** 重新开始。我军全灭则新开一局。侧栏显示 **背景提要** 与 **胜利条件** 摘要。

### 秘籍（选关 · 将领图鉴 · 易忘备忘）

> 开发/试玩用，不参与存档逻辑。

- **Ctrl + Shift + K**：打开 **「秘籍 · 选关」** 面板，列出全部关卡；**点击某一关** 直接进入该关（默认刘关张开局数值）。
- **Ctrl + Shift + J**：打开 **「将领图鉴」**（**100+** 演义人物），支持搜索；**点击姓名** 查看列传与图鉴基准属性（与单场战斗内等级缩放后的数值可能不同）。
- **Esc** 或 **再按一次** 对应组合键关闭面板；焦点在输入框内时 **不会触发** 秘籍（图鉴内搜索框除外）。

### 回合与胜负

- **回合制**：先完整进行「我军回合」，再进入「敌军回合」；敌军行动由简单 AI 控制（靠近我军、进入攻击范围则攻击）。
- **胜利**：达成本关 **胜利条件**（全歼或歼灭指定敌方单位）；见侧栏「胜利条件」说明。
- **失败**：我军全部击退。

### 移动与战斗

- 点击**尚未完成本回合行动**的我军单位：显示**蓝色**可移动范围（四向、不可穿人、受移动力限制），再点击蓝色格子完成移动。
- **不想移动**：在移动阶段**再次点击同一武将**，或**点击该武将脚下的格子**，会直接进入行动菜单（视为原地待命，`moved` 已消耗）。
- 移动结束或原地待命后，棋子旁弹出**行动菜单**：**发动攻击**、**使用计策**、**待机**。
  - **发动攻击** / **使用计策**：条件与此前相同（贴邻 / 距离 ≤2）。若范围内**只有一名**敌军则直接结算；若**多名**则进入**目标选择**：可用**方向键**切换高亮敌军、**Enter** 确认，或**鼠标指向**同步高亮、**点击**敌军确认。
  - **待机**：始终可选，结束该单位本回合行动。
- 菜单可用**鼠标指向高亮项并点击**，或使用**方向键**与 **Enter**；侧栏 **「待机」** 在移动前/菜单中均可结束当前选中单位的行动。
- **Esc** 或 **鼠标右键**：在**目标选择**阶段返回上一层菜单；在**行动菜单**阶段则**取消本武将本回合操作**，将其**位置与 moved/acted 恢复到我军回合开始时**的快照（未移动未行动则回到选中前状态）。
- 尚未移动时也可点侧栏 **「待机」**，视为原地结束该单位行动（移动与攻击均跳过）。
- 单位**扣血**（未阵亡）时有抖动、红光与飘字伤害数字；**移动**时棋子从上一格滑入新格；**阵亡**时原地短暂保留虚影并渐隐消失，战场顶部会显示 **「{姓名}被斩于阵前」**（敌我均适用）。战报区同步文字记录。

### 其他界面功能

- **新游戏**：重置为序章初始局面。
- **战报**：显示最近若干条战场日志。
- 存档与账号相关说明见下文「存档说明」。

## 环境要求

- [Node.js](https://nodejs.org/)（建议 LTS）

## 开发运行

在项目根目录执行：

```bash
npm install
npm run dev
```

- 前端开发服务器：<http://localhost:5173>
- 后端 API：<http://localhost:8787>  
  开发时前端的 `/api` 请求会通过 Vite 代理到后端。

## 生产构建与预览

先构建前端，再由 Express 托管静态文件并与 API 同端口：

```bash
npm run preview
```

浏览器访问：<http://localhost:8787>

## 配置

- 复制 `server/.env.example` 为 `server/.env`（可选；仓库内已有开发用默认值）。
- 修改 `PORT`（默认 `8787`）、`JWT_SECRET`（**正式环境务必改为足够长的随机字符串**）。

## 存档说明

- **本地**：保存在浏览器 `localStorage`，不登录也可使用。
- **云端**：登录后写入服务器，数据文件为 `server/data/store.json`（目录已用 `.gitignore` 忽略内容，仅保留占位文件）。
