# 三国英杰传 · 网页版

单机战棋试玩、本地/云端存档与账号系统（注册登录 + JWT）。

## 已实现玩法

### 关卡与单位

- 当前关卡：**序章 · 讨伐黄巾**（`scenarioId: prologue_zhangjiao`）。
- 战场为 **12×8** 格四向网格。
- **我军**：刘备、关羽、张飞（各有 HP、攻击力、移动力等属性）。
- **敌军**：两名黄巾贼、一名黄巾头目。

### 回合与胜负

- **回合制**：先完整进行「我军回合」，再进入「敌军回合」；敌军行动由简单 AI 控制（靠近我军、进入攻击范围则攻击）。
- **胜利**：敌军全部击退（HP 归零）。
- **失败**：我军全部击退。

### 移动与战斗

- 点击**尚未完成本回合行动**的我军单位：显示**蓝色**可移动范围（四向、不可穿人、受移动力限制），再点击蓝色格子完成移动。
- 移动结束后，棋子旁弹出**行动菜单**：**发动攻击**、**使用计策**、**待机**。
  - **发动攻击**：与任意存活敌军**贴邻**（上下左右一格）时为亮色，否则灰色不可选；确认后对贴邻敌军中 **HP 最低**的一体造成全额攻击伤害。
  - **使用计策**：与任意存活敌军**曼哈顿距离 ≤ 2**（含贴邻与隔一格）时为亮色，否则灰色；确认后对范围内 **HP 最低**的一体造成约 **55% 攻击力**的法术伤害。
  - **待机**：始终可选，结束该单位本回合行动。
- 菜单可用**鼠标指向高亮项并点击**，或使用**方向键（上下左右）**在可选项间循环，**Enter** 确认；侧栏 **「待机」** 在移动前/菜单中均可结束当前选中单位的行动。
- 尚未移动时也可点侧栏 **「待机」**，视为原地结束该单位行动（移动与攻击均跳过）。
- 单位**扣血**时会有抖动、红光与飘字伤害数字等简短动画；战报区同步记录。

### 其他界面功能

- **新游戏**：重置为序章初始局面。
- **战报**：显示最近若干条战场日志。
- 存档与账号相关说明见下文「存档说明」。

## 环境要求

- [Node.js](https://nodejs.org/)（建议 LTS）

## 开发运行

在项目根目录执行：

```bash
npm install
npm run dev
```

- 前端开发服务器：<http://localhost:5173>
- 后端 API：<http://localhost:8787>  
  开发时前端的 `/api` 请求会通过 Vite 代理到后端。

## 生产构建与预览

先构建前端，再由 Express 托管静态文件并与 API 同端口：

```bash
npm run preview
```

浏览器访问：<http://localhost:8787>

## 配置

- 复制 `server/.env.example` 为 `server/.env`（可选；仓库内已有开发用默认值）。
- 修改 `PORT`（默认 `8787`）、`JWT_SECRET`（**正式环境务必改为足够长的随机字符串**）。

## 存档说明

- **本地**：保存在浏览器 `localStorage`，不登录也可使用。
- **云端**：登录后写入服务器，数据文件为 `server/data/store.json`（目录已用 `.gitignore` 忽略内容，仅保留占位文件）。
